Python 3.8 or higher Tested on 3.9 and 3.10

Installation

1. Clone the Repository

git clone https://github.com/mohamedtarek750/Spark-MongoDB-Project.git
cd Spark-MongoDB-Project

2. Install Python Dependencies

pip install -r requirements.txt

This installs pyspark and pymongo, which are the only two direct dependencies.

3. Verify Java Installation

java -version
# Expected: openjdk version "11.x.x" or "1.8.x_xxx"
Note

If Spark fails to start, ensure the JAVA_HOME environment variable is set correctly and points to a Java 8 or 11 installation.

Configuration

Two values must be updated in spark-py.py before running: the MongoDB connection URI and the local dataset paths.

MongoDB Connection URI

Replace the hardcoded credentials with your own MongoDB Atlas connection string:

client = MongoClient(
    "mongodb+srv://<username>:<password>@<cluster>.mongodb.net/?appName=Cluster0"
)

The project connects to a database named spark_mango and writes to three collections: customers, people, and organizations.

Dataset Paths

Update the CSV file paths to match your local file system:

customers     = spark.read.csv("/path/to/customers-10000.csv", header=True)
people        = spark.read.csv("/path/to/people-10000.csv",   header=True)
organizations = spark.read.csv("/path/to/organizations-10000.csv", header=True)

Recommended: Use Environment Variables

For security, load the connection URI from the environment rather than hardcoding it:

import os
from pymongo import MongoClient

client = MongoClient(os.environ["MONGO_URI"])

Then set it in your shell before running:

export MONGO_URI="mongodb+srv://<username>:<password>@<cluster>.mongodb.net/"
python spark-py.py

Pipeline

The pipeline runs sequentially through nine stages. Each stage is self-contained and clearly commented in spark-py.py.

01

Create Spark Session

Initializes a local SparkSession with the application name SparkMongoProject. This is the entry point for all Spark operations.

02

Load Data

Reads three CSV files into Spark DataFrames with header=True. Spark automatically infers column names from the first row of each file.

03

Data Preparation

Removes duplicate rows from all three DataFrames using .dropDuplicates(). Additionally, the Subscription Date column in the customers DataFrame is cast from string to a proper DateType using the format yyyy-MM-dd and stored as subscription_date_fixed.

04

Basic Exploration

Prints row counts for all three DataFrames to stdout. Provides a quick sanity check that data loaded correctly and deduplication had the expected effect.

05

Connect to MongoDB Atlas

Establishes a connection to a MongoDB Atlas cluster using PyMongo's MongoClient. Points to the spark_mango database and references three collection handles: customers, people, organizations.

06

Convert to JSON

Converts each Spark DataFrame to a list of JSON strings using .toJSON().collect(). Each row becomes a valid JSON object that can be directly deserialized by PyMongo.

07

Insert into MongoDB

Clears all three collections with delete_many({}) to ensure idempotency, then bulk-inserts the JSON documents using insert_many(). This guarantees a clean state on every run.

08

MongoDB Aggregation Queries

Runs four analytical aggregation pipelines directly against the customers collection. Results are printed to stdout. See the Queries section for full details.

09

Stop Spark

Gracefully shuts down the SparkSession with spark.stop(), releasing all resources and JVM threads.

MongoDB Queries

Four aggregation pipelines are executed against the customers collection after data ingestion. All queries use MongoDB's $group, $sort, and $limit operators.

Customers per Country
Groups all customer records by the Country field, counts occurrences, and sorts descending by count.
Top 5 Countries
Same as above, but applies a $limit: 5 stage to return only the five most represented countries.
Customers per Company
Groups by the Company field to identify which organizations have the highest customer concentration.
Subscription Date Range
Uses $min and $max accumulators in a single group stage to return the earliest and latest subscription dates across all customers.

Example: Customers per Country

customers_col.aggregate([
    { "$group": { "_id": "$Country", "count": { "$sum": 1 } } },
    { "$sort":  { "count": -1 } }
])

Example: Subscription Date Range

customers_col.aggregate([
    {
        "$group": {
            "_id":     None,
            "earliest": { "$min": "$Subscription Date" },
            "latest":   { "$max": "$Subscription Date" }
        }
    }
])

Security

Warning

The original spark-py.py file contains a hardcoded MongoDB Atlas connection string including credentials. These should be considered compromised and rotated immediately in your MongoDB Atlas dashboard.

Recommended Remediation

  • Rotate the MongoDB Atlas database user password via the Atlas console under Database Access
  • Remove or redact the credentials from the commit history using git filter-repo or BFG Repo Cleaner
  • Store the URI in an environment variable or a .env file (excluded via .gitignore)
  • Use MongoDB Atlas IP Access Lists to restrict which IPs can connect to the cluster

Secure Pattern

# .env (never commit this file)
MONGO_URI=mongodb+srv://user:pass@cluster.mongodb.net/

# spark-py.py
import os
from pymongo import MongoClient
from dotenv import load_dotenv

load_dotenv()
client = MongoClient(os.environ["MONGO_URI"])

Install python-dotenv to load .env files automatically:

pip install python-dotenv
